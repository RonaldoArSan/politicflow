# PoliticFlow Project Instructions

Este arquivo contém convenções específicas do projeto PoliticFlow, um SaaS multi-tenant para gestão política, eleitoral e de gabinete.

## Arquitetura Geral

- **Next.js App Router** como framework principal
- **API-first** design para futura reutilização em mobile
- **Monólito modular** no MVP, evitando microserviços desnecessários
- **PostgreSQL** como banco de dados principal
- **Prisma** como única camada de acesso ao banco
- **JWT + Refresh Token** para autenticação
- **RBAC** obrigatório em toda rota protegida
- **Multi-tenancy** por isolamento lógico com `tenantId`
- **Soft delete** com campo `deletedAt`
- **Auditoria obrigatória** em operações críticas

### Route Handlers

**Localização**: `src/app/api/**/route.ts`

**Padrão obrigatório**:
```typescript
import { withAuth, apiResponse, apiError, auditLog } from '@/lib/api-helpers';
import { tenantWhere } from '@/lib/tenant';
import { z } from 'zod';

const createSchema = z.object({
  name: z.string().min(1),
  // ... outros campos
});

async function handleGET(req: Request, tenantId: string, user: User) {
  const { searchParams } = new URL(req.url);
  const page = parseInt(searchParams.get('page') || '1');
  const limit = parseInt(searchParams.get('limit') || '20');
  const skip = (page - 1) * limit;

  const data = await prisma.model.findMany({
    where: tenantWhere(tenantId),
    skip,
    take: limit,
    orderBy: { createdAt: 'desc' }
  });

  return apiResponse(data);
}

async function handlePOST(req: Request, tenantId: string, user: User) {
  const body = await req.json();
  const validatedData = createSchema.parse(body);

  const result = await prisma.model.create({
    data: {
      ...validatedData,
      tenantId
    }
  });

  await auditLog(tenantId, user.id, 'model', 'CREATE', result.id, null, result);

  return apiResponse(result, 'Criado com sucesso', 201);
}

export const GET = withAuth(handleGET, { module: 'models', action: 'read' });
export const POST = withAuth(handlePOST, { module: 'models', action: 'create' });
```

### Server Actions

- **Uso limitado**: Apenas para operações simples que não precisam de regras críticas
- **Priorizar API**: Para autenticação, permissões, validações complexas, auditoria
- **Quando usar**: Formulários simples, operações idempotentes sem impacto crítico

## Prisma (Camada de Dados)

### Schema Conventions

```prisma
model Committee {
  id        String   @id @default(cuid())
  tenantId  String
  name      String
  slug      String   @unique(["tenantId", "slug"])
  parentId  String?
  status    CommitteeStatus @default(ACTIVE)

  tenant    Tenant   @relation(fields: [tenantId], references: [id], onDelete: Cascade)
  parent    Committee? @relation("CommitteeHierarchy", fields: [parentId], references: [id])
  children  Committee[] @relation("CommitteeHierarchy")

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  deletedAt DateTime?

  @@map("committees")
}
```

**Regras obrigatórias**:
- Todo modelo tem `tenantId` (exceto tabelas globais)
- Campos `createdAt`, `updatedAt`, `deletedAt` em todos os modelos
- Soft delete: queries devem filtrar `deletedAt IS NULL`
- Índices compostos para `tenantId` + campos únicos
- Relacionamentos com `onDelete: Cascade` para tenant

### Query Patterns

```typescript
// Sempre filtrar por tenant
const committees = await prisma.committee.findMany({
  where: {
    ...tenantWhere(tenantId),
    deletedAt: null, // Soft delete
    status: 'ACTIVE'
  }
});

// Para updates/deletes
await prisma.committee.updateMany({
  where: {
    ...tenantWhere(tenantId),
    id: committeeId,
    deletedAt: null
  },
  data: { deletedAt: new Date() } // Soft delete
});
```

## Multi-Tenancy

### Tenant Context

```typescript
// lib/tenant.ts
export function tenantWhere(tenantId: string) {
  return { tenantId };
}

export async function getTenantFromRequest(req: Request): Promise<Tenant> {
  const token = getTokenFromRequest(req);
  const payload = verifyToken(token);
  const tenant = await prisma.tenant.findUnique({
    where: { id: payload.tenantId }
  });

  if (!tenant || tenant.status !== 'ACTIVE') {
    throw new Error('Tenant not found or inactive');
  }

  return tenant;
}

export async function validateUserInTenant(userId: string, tenantId: string): Promise<User> {
  const user = await prisma.user.findFirst({
    where: {
      id: userId,
      tenantId,
      deletedAt: null
    }
  });

  if (!user) {
    throw new Error('User not found in tenant');
  }

  return user;
}
```

## Auditoria

### Padrão de Auditoria

```typescript
// lib/audit.ts
export async function auditLog(
  tenantId: string,
  userId: string,
  entity: string,
  action: string,
  entityId: string,
  oldData?: any,
  newData?: any,
  req?: Request
) {
  const ip = req ? getClientIP(req) : null;
  const userAgent = req ? req.headers.get('user-agent') : null;

  await prisma.auditLog.create({
    data: {
      tenantId,
      userId,
      entity,
      action,
      entityId,
      oldData: oldData ? JSON.stringify(oldData) : null,
      newData: newData ? JSON.stringify(newData) : null,
      ip,
      userAgent
    }
  });
}
```

## Services e Repositories

### Services para Regras de Negócio

```typescript
// lib/services/committee-service.ts
export class CommitteeService {
  static async create(data: CreateCommitteeData, tenantId: string, userId: string) {
    // Validações de negócio
    await this.validateCommitteeLimits(tenantId);
    await this.validateParentCommittee(data.parentId, tenantId);

    // Criação
    const committee = await prisma.committee.create({
      data: { ...data, tenantId }
    });

    // Auditoria
    await auditLog(tenantId, userId, 'committee', 'CREATE', committee.id, null, committee);

    return committee;
  }

  private static async validateCommitteeLimits(tenantId: string) {
    const count = await prisma.committee.count({
      where: tenantWhere(tenantId)
    });

    const plan = await getTenantPlan(tenantId);
    if (count >= plan.maxCommittees) {
      throw new Error('Limite de comitês atingido');
    }
  }
}
```

## Convenções de Código

### Nomes

- **Arquivos**: kebab-case (`committee-form.tsx`, `api-helpers.ts`)
- **Componentes**: PascalCase (`CommitteeForm`, `DataTable`)
- **Funções/Variáveis**: camelCase (`createCommittee`, `validateUser`)
- **Constantes**: UPPER_SNAKE_CASE (`MODULES`, `ACTIONS`)
- **Types**: PascalCase (`CommitteeData`, `ApiResponse`)
- **Database/Prisma**: PascalCase models, camelCase fields

### Textos

- **Código**: Inglês (variáveis, comentários, funções)
- **UI**: Português-BR (labels, mensagens, placeholders)

### Segurança

- **Nunca expor dados sensíveis** no frontend
- **Validar tudo no backend** (não confiar no frontend)
- **Usar prepared statements** (Prisma já faz isso)
- **Sanitizar inputs** (Zod ajuda)
- **Rate limiting** em endpoints críticos
- **Logs de segurança** para tentativas suspeitas

### Preparação para Mobile

- **API-first**: Toda lógica crítica no backend
- **Endpoints RESTful**: Padrão consistente
- **Paginação**: Mesmo padrão para web e mobile
- **Autenticação**: JWT preparado para mobile
- **Validações**: Mesmo schema para web e mobile
- **Respostas**: Formato consistente JSON
